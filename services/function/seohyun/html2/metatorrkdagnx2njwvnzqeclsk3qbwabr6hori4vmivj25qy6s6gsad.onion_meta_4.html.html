Вкурил https://telegra.ph/Kak-otkryt-darknet-parashu-posobie-dlya-domohozyaek-03-28, возникли замечания:

1. Вместо указания в конфиге nginx нескольких блоков

server {}

, различающихся только директивами

listen

и

server_name

, проще в один блок

server {}

насыпать все

listen

'ы, а все

server_name

'ы указать одной строкой, wildcard'ом, или вовсе не указывать. Также не вижу смысла повторять дефолты и грузить бедных домохозяек разделением конфига на

include

'ы. Весь конфиг сайта, аналогичный имеющемуся в гайде, может выглядеть так:

server {
# I2P service
listen 127.0.0.1:8082;
# onion service
listen 127.0.0.1:8081;
root /var/www/plus33;
charset utf-8;
}

2. В конфиге nginx отсутствуют директивы обращения к динамике, в т.ч. скриптам постинга. Мне кажется, домохозяйки поверят заголовку и будут надеяться на полноту гайда.

3. Для различения пользователей tor можно вывести идентификатор цепочки в виде ipv6-адреса клиента, добавив в torrc

HiddenServiceExportCircuitID haproxy

и дописав

proxy_protocol

в соответствующую директиву

listen

nginx'а. Как аналогичное сделать в i2p, я не знаю.

4. В качестве места для замечаний в гайде указаны клирнетовские ресурсы, которые из тора не пускают без прохождения проверок cloudflare.

Ну и соснули по теме гайда, с которыми мне нужна помощь. Просунув i2pd через тор, столкнулся с тем, что спустя час или два аптайма i2pd почти стабильно теряет способность строить цепочки. Пока что настроил его перезапуск каждый час, но хотелось бы этого избежать.

3 поста и 1 изображение пропущено.
